Launching Hugo website on Github

2020-06-04

Publishing a website often requires you to buy a domain name first. A domain name is like having a real estate online. For example, if you bought the domain “yourwebsite.com” you have complete control of what is being published on that domain and its corresponding ‘sub’-URLs, such as “yourwebsite.com/about”. Even though most domain names are not very expensive it can be a bit intimidating to buy one. However, there are alternatives! GitHub does offer a service that allows you to publish a website without needing to buy a domain name, namely via GitHub Pages. GitHub Pages is a static web hosting service which provides 1GB of free space. The only requirement you have to fulfil is having a GitHub account. Another nice feature is that GitHub pages also allows you to publish a website without needing to use or know HTML because you can use a built-in theme alternatively to HTML code. However, if you want to use your own HTML rendered website than that is also possible.

I created my website with the help of a Hugo template (the “Hugo Theme Minos") and therefore, I launched it on GitHub without using a GitHub theme with the following steps:

Launching your website for the first time

  1. Modify the base URL link in the “config.toml” file within your website folder to:
baseURL = "https://<YOUR-USER-NAME>.github.io/"
  1. Create git repository .github.io. This repository is initially emtpy but will later contain the content for your static rendered website.
  2. Build your website by running the following command at your website folder:
hugo

This command builds your website in a public folder that is located inside your website folder. Within the public folder there is a index.html file which is required by GitHub to create a website.

If you want to check the layout and content of your website first you should run:

hugo server

This enables you to access your website via http://localhost:1313. Don’t forget to close the hugo server before creating your public folder.

  1. Go to your public folder which contains webfiles, such as HTML, css,…, that can generate your static website.
  2. Run the following commands to add you public folder to your <USERNAME>.github.io repository.
cd public
git init
git add .
git remote add origin https://github.com/username/username.github.io.git
git commit -m “first commit”
git push origin master

The first command changes reference directories to your public folder. If you already opened a Terminal at the public folder then you can leave this command out. After running this command you should see that your content has been added to your git repository at ‘https://github.com/username/username.github.io.git'.

  1. In setting you should now see a green box which informs you that your website has been published under the domain ‘https://.github.io/'.

If you need further guidance check out this post.

Updating your content

After having launched the website new content can be updated by simply commiting it to the GitHub repository, like you would do for any other Git project.

  1. First, you need to update the public folder by running the hugo command in the command line of your main website folder.
  2. Add the new content to Git, commit it and push to your git repository by running the following commands:
cd public
git add
git commit -m "YOUR-COMMIT-MESSAGE"
git push -u origin master
